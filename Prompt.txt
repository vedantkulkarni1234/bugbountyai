## PHASE 1: PASSIVE INTELLIGENCE GATHERING

### 1.1 Domain & Infrastructure Analysis
- **DNS Enumeration**: Extract all DNS records (A, AAAA, MX, TXT, NS, SOA, CAA)
- **Subdomain Discovery**: Identify all subdomains using:
  - Certificate transparency logs (crt.sh, Censys)
  - Search engine dorking (Google, Bing, Shodan)
  - DNS brute-forcing wordlists
  - Reverse DNS lookups
- **ASN & IP Range Mapping**: Identify autonomous system numbers and associated IP blocks
- **WHOIS Analysis**: Historical ownership, registrar info, contact details

### 1.2 Technology Stack Fingerprinting
- **Web Technologies**: Identify frameworks, CMS, programming languages
- **Server Infrastructure**: Web servers, load balancers, CDN usage
- **JavaScript Libraries**: Frontend frameworks, third-party scripts
- **Security Headers**: Analyze CSP, HSTS, X-Frame-Options, etc.
- **Cookies & Session Management**: Token patterns, security flags

### 1.3 Digital Footprint Analysis
- **Search Engine Reconnaissance**:
  - Google Dorks: `site:tradepmr.com filetype:pdf/xlsx/doc`
  - Exposed documents, backup files, configuration files
  - Employee emails and organizational structure
- **Code Repositories**: GitHub, GitLab, Bitbucket searches for:

  - Source code leaks
  - Internal documentation

### 1.4 Third-Party Integrations
- **API Endpoints**: Discover external service integrations
- **OAuth/SSO Providers**: Authentication mechanisms
- **Payment Gateways**: PCI-DSS relevant components
- **Analytics & Tracking**: Google Analytics, marketing pixels

## PHASE 2: ACTIVE RECONNAISSANCE

### 2.1 Attack Surface Mapping

- **Parameter Analysis**: Query parameters, POST data structures

### 2.2 Application Behavior Analysis

### 2.3 Security Mechanism Identification

- **Bot Protection**: CAPTCHA, fingerprinting, behavioral analysis
- **Encryption**: TLS/SSL versions, cipher suites, certificate validation
- **CORS Policies**: Cross-origin resource sharing configurations
